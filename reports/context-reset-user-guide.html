1. Overview

context-reset manages Claude Code session lifecycle across terminals. When a session's context window fills up, it seamlessly transfers work to a fresh Claude instance in a new terminal tab — no human intervention needed.

FeatureDetails
Two scriptscontext_reset.py (kills old tab) and new_session.py (keeps old tab)
State handoffSESSION_STATE.md (auto) + TODO.md (manual)
Tab UXPer-project colors, folder name titles, no focus steal
SafetyTwo-phase verification, multi-Claude guard, duplicate session guard, kill retry, audit logging
DependenciesPython 3.8+ stdlib only. No pip dependencies.

2. Quick Start

1 Install

pip install git+https://github.com/grobomo/context-reset

2 Tell Claude about it — add to your project's CLAUDE.md:

## Context Reset
When context gets long, save your progress to TODO.md and run:
  context-reset --project-dir $CLAUDE_PROJECT_DIR

3 Done. Claude will run context-reset when context fills up, carrying over TODO.md and a readable conversation summary.

For fully autonomous operation (Claude loops through tasks endlessly), install hook-runner: npx grobomo/hook-runner --yes. Its auto-continue Stop module blocks every stop and tells Claude to keep working. When context fills up, Claude calls context-reset itself. See the Integration section in the Admin Reference for the full architecture.
Works on Windows, WSL, macOS, and Linux with no extra configuration. WSL is auto-detected and routes through Windows Terminal.

4. Usage Examples

Context Reset (kills old tab)

context-reset --project-dir /path/to/project        # pip CLI
python context_reset.py --project-dir /path/to/project  # from source

New Session (keeps old tab)

new-session --project-dir /path/to/project
new-session --project-dir /current --target-project /other
new-session --prompt "Fix the failing tests"

Preview / Debug

context-reset --dry-run     # show command without executing
new-session --dry-run       # same, keeps old tab

5. How It Works

1 Extract state — Reads last 500 JSONL lines from transcript, writes readable SESSION_STATE.md (~8K tokens)

2 Write prompt — Prompt saved to .claude-next-prompt file (avoids shell escaping on all platforms)

3 Launch tab — Platform-specific: wt (Win/WSL), osascript (Mac), gnome-terminal/tmux (Linux)

4 Verify — Wait for new Claude process, then confirm transcript file activity

5 Kill old tab — context_reset.py only. Walks process tree, refuses if multiple Claudes attached

6. Troubleshooting

ProblemSolution
Trust dialog on new tabScript pre-trusts via ~/.claude.json. Check file is writable.
Tab doesn't open (Windows)Verify wt --version works from terminal.
Tab doesn't open (WSL)WSL interop must be enabled. Check wt.exe --version from WSL.
Focus stealFixed: focus-tab --previous (WT tab) + Win32 GetForegroundWindow/SetForegroundWindow (OS window).
Semicolons break launchFixed in v1.1.0: prompt-file approach on all platforms.
Old tab not killedFixed: taskkill now captures stderr, retries once, verifies PID death. Run --dry-run to check PID detection.
Duplicate tabsFixed: pre-launch guard checks if target project has an active transcript (<10s old). Refuses to spawn duplicates.
